Privacy Policy

ToolKit – seamless file & image tools, zero data footprint
📅 Last updated: November 18, 2025

ToolKit (the "app", "we", "us", or "our") provides a collection of browser‑based utilities for image editing, PDF conversion, compression, background removal, and other file operations. This privacy policy explains how we safeguard your information when you use our web tools or the Android version of the app, hosted at raja1068.github.io/Tools. We believe that privacy is not just a legal requirement but a fundamental right — so we designed ToolKit with zero data retention.

1. Information we never collect

We have built ToolKit to respect your digital autonomy. We do not collect, store, share, or sell any personally identifiable information, including but not limited to:

  • Names, email addresses, or phone numbers
  • Location data, IP addresses (no server-side logging of user IPs)
  • Device identifiers, advertising IDs, or fingerprinting data
  • Contact lists, calendar, or sensor data

Your activity stays entirely private — there are no analytics trackers, no telemetry, and no third-party monitoring scripts except standard AdSense placements for monetization, which are bound by their own policies. AdSense does not receive any personal user data from ToolKit's internal logic.

2. How file processing works (temporary & private)

All tools operate with a strict “process-and-forget” principle. When you upload or open a file for conversion, compression, background removal, or editing:

  • Client-side priority: Many transformations happen directly in your browser using WebAssembly, Canvas, or JavaScript libraries. Files never leave your device unless required by the specific tool (e.g., server‑assisted PDF merging or advanced OCR).
  • Server-assisted processing: For certain operations, the file is transmitted via encrypted HTTPS to our temporary processing endpoint or a trusted third‑party API (e.g., background removal or high‑resolution conversion). The file remains in memory only for the duration of the task — typically a few seconds. After the result is delivered, the original file and any intermediate copies are instantly purged from all servers. No backups, no logging, no caching.
  • No file is ever stored for more than the necessary processing window. We do not use your files for training, analysis, or any form of data mining.

All communications are protected with TLS 1.2+ ensuring secure transit.

3. Permissions & local access (Android & Web)

For Android App version: ToolKit requests minimal permissions:

  • INTERNET — to connect to the online toolkit and perform file operations.
  • ACCESS_NETWORK_STATE — to verify network connectivity and optimize performance.

ToolKit does not request storage, media, camera, microphone, contacts, or location permissions. On the web platform, when you choose a file using the system file picker, that file is not read unless you explicitly activate a tool; the browser handles the selection securely. No background access occurs.

4. Third-party services & API usage

Some tools may rely on trusted external services to provide advanced functionality (e.g., image background removal or enhanced PDF compression). Each third-party vendor is carefully vetted for compliance with data protection standards, and they are contractually bound to delete files immediately after processing. Third parties do not receive any identifiable user information.

The only external script included is Google AdSense (for showing non-intrusive ads to support ToolKit's free operation). AdSense may use cookies or device information for serving relevant ads according to Google's Privacy Policy. You can manage ad personalization via Google’s Ad Settings. ToolKit does not control or access any data collected by Google.

5. No accounts · No logins · No tracking

We don't believe in forcing users to create accounts to edit an image or convert a document. ToolKit works fully offline-ready (where possible) and requires no sign-up, registration, or subscription. Consequently, we do not store any user profiles, passwords, or authentication tokens. There is no mechanism to “track” your activity across sessions — every visit is anonymous.

6. Children's privacy & COPPA compliance

ToolKit is designed for general audiences and complies with the Children's Online Privacy Protection Act (COPPA) and similar international regulations. We do not knowingly collect any personal information from children under 13 years of age. Since we collect absolutely no data, there is no risk of accidental collection of child-specific information. Parents and guardians can be confident that using ToolKit does not expose minors to data harvesting.

7. Data security measures

We implement robust security practices: end-to-end encryption during file transmission, automatic file deletion after processing, and regular audits of our infrastructure. While no internet transmission is 100% foolproof, we constantly update dependencies, enforce HTTPS Strict Transport Security (HSTS), and maintain a “privacy by design” approach. In the unlikely event of a security incident affecting user data, we would follow all applicable disclosure requirements — though, we reiterate, we store nothing.

8. Your rights & international privacy

Because ToolKit does not collect or store personal information, rights such as access, rectification, erasure, and data portability (under GDPR or CCPA) are inherently fulfilled. If you have any residual questions or want to verify our claims, you may contact us directly using the details below. We will respond within 30 days.

9. Changes to this Privacy Policy

We may update this policy to reflect improvements, regulatory updates, or new tool features. The “Last updated” date at the top of this page indicates when the latest revision was made. If significant changes occur, we will highlight them on the ToolKit homepage. Your continued use of our tools after any modification constitutes acceptance of the updated policy. However, we will never retroactively change how we handle your data.
